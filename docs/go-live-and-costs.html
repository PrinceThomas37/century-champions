Century Steel Profiles

Century Champions — Go-Live Plan & Costs

Contractor rewards programme · prepared for the owner · everything owned under the client's accounts

The app is already built and running on professional cloud infrastructure. Going live means swapping the test pieces for real ones and handing ownership to the client. Below is the step-by-step plan and the full budget. Start with Part 1 (SMS) — it has a 1–2 day approval wait; everything else can run in parallel.

Owner = client does · Dev = developer does · Both = quick handoff together

Part A — Go-Live Checklist

1 · SMS / OTP login (do first — has a waiting period)

  1. Owner Subscribe to the OTP SMS vendor.
  2. Owner Complete DLT registration (legally required in India). Register a Sender ID (e.g. CENTRY) and a template:
    "##otp## is your Century Champions login code. Do not share it with anyone." (variable named otp)
  3. Owner Collect from vendor: API key, Template ID, Sender ID, API docs.
  4. Owner → Dev Send those to the developer to wire in.

2 · Create accounts (owner's email + owner's card)

  1. Owner Create a GitHub account (free).
  2. Owner Create Supabase account + organisation → upgrade to Pro, add card.
  3. Owner Create Vercel account → upgrade to Pro, add card.

3 · Transfer ownership to the client

  1. Both Transfer the code repository to the owner; add developer back as collaborator.
  2. Both Move the database into the owner's account (developer migrates the data).
  3. Both Move the hosting to the owner's account (developer reconnects settings).

4 · Domain (no new purchase)

  1. Owner Add one DNS record for a subdomain like rewards.centurysteelprofiles.com (developer provides the exact value). Main website is unaffected.

5 · Real data (owner provides, developer loads)

  1. Owner → Dev Product list + points per packet.
  2. Owner → Dev Reward tiers (thresholds + rewards).
  3. Owner → Dev The real serial codes printed on packets (a file).

6 · Launch

  1. Dev Wire in real SMS, load data, rotate all secrets, point domain, switch off test mode.
  2. Owner Test on your phone — real SMS code, log in, scan a code.
  3. Dev Generate QR codes for the live link.
  4. Owner Print & place QR codes at distributor counters. Live.
